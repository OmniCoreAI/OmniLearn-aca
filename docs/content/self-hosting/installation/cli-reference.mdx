# CLI Reference

Complete reference for all OmniLearn CLI commands. The CLI is published as `omnilearn` on npm and is typically run via `npx omnilearn@latest`.

## Setup and Configuration

### `omnilearn setup`

Run the interactive setup wizard. This command generates `docker-compose.yml`, `.env`, `omnilearn.config.json`, and nginx or Caddy configuration files based on your choices.

```bash
omnilearn setup
```

Flags: `--ci`, `--name`, `--domain`, `--port`, `--admin-email`, `--admin-password`, `--channel <stable|dev>`, `--no-start`.

You can re-run this command at any time to reconfigure your instance. Existing configuration will be preserved as defaults in the prompts.

### `omnilearn config`

View the current configuration.

```bash
omnilearn config
```

### `omnilearn env`

Interactively edit environment variables in your `.env` file (Domain & Hosting, Database & Redis, Security, AI, Email (Resend), S3 Storage, OAuth).

```bash
omnilearn env
```

<Callout type="info">
The `env` command edits a curated set of variables only. SMTP credentials (`OMNILEARN_SMTP_*`) are not editable through this command — edit `.env` directly if you need them.
</Callout>

## Service Management

### `omnilearn start`

Start all OmniLearn services. On first run, this pulls the required Docker images before starting containers.

```bash
omnilearn start
```

### `omnilearn stop`

Stop all running OmniLearn services. Data is preserved in Docker volumes.

```bash
omnilearn stop
```

### `omnilearn status`

Show the status of each OmniLearn service.

```bash
omnilearn status
```

### `omnilearn logs`

Stream container logs for all services.

```bash
omnilearn logs
```

### `omnilearn update`

Update OmniLearn to the latest release or a specific version. Pulls the target image, rewrites `docker-compose.yml`, restarts services, and prompts for database migrations.

```bash
omnilearn update
omnilearn update --version 1.4.2
omnilearn update --migrate       # auto-apply migrations
omnilearn update --no-migrate    # skip migrations
```

### `omnilearn dev`

Start a full development environment (PostgreSQL + Redis in Docker, API + web + collab run locally).

```bash
omnilearn dev
```

## Diagnostics

### `omnilearn doctor`

Run diagnostics on your OmniLearn installation. Checks Docker availability, container health, port conflicts, DNS resolution, and disk space.

```bash
omnilearn doctor
```

### `omnilearn health`

Run the CLI's health checks (PostgreSQL connection, Redis ping, HTTP health endpoint, disk, resources).

```bash
omnilearn health
```

### `omnilearn shell`

Open an interactive shell inside a running container for debugging.

```bash
omnilearn shell
```

## Infrastructure

### `omnilearn deployments`

Manage deployments and scale memory limits for your OmniLearn containers.

```bash
omnilearn deployments
```

## Backup and Restore

### `omnilearn backup`

Create a PostgreSQL dump archive of your instance's database (plus a copy of `.env`). Archives are written to `<installDir>/backups/`.

```bash
omnilearn backup
```

### `omnilearn restore`

Restore a database from a backup archive.

```bash
omnilearn restore <path/to/omnilearn-backup-<timestamp>.tar.gz>
```
